Control loop — what happens, in order

  1. 1Route 53 resolves the client's request and routes it to the Application Load Balancer.
  2. 2Traffic enters the VPC through the Internet Gateway and reaches the ALB in the public subnets.
  3. 3The ALB distributes requests across healthy targets and continuously runs target-group health checks against every instance in the Auto Scaling Group.
  4. 4A failed health check marks the instance Unhealthy. The ASG schedules it for replacement, firing the EC2_INSTANCE_TERMINATING lifecycle hook and moving it to Terminating:Wait.
  5. 5EventBridge routes the lifecycle event to the Terminate Handler Lambda.
  6. 6The Lambda deregisters the instance from the target group, drains in-flight connections, and archives logs and session state to S3.
  7. 7The Lambda calls CompleteLifecycleAction(CONTINUE); the ASG terminates the old instance and launches a replacement from the current Launch Template.
  8. 8The new instance fires EC2_INSTANCE_LAUNCHING (Pending:Wait); EventBridge routes this event to the Launch Handler Lambda.
  9. 9The Lambda triggers an SSM Automation runbook to bootstrap the instance and run application readiness checks, then completes the lifecycle action.
  10. 10The instance passes health checks, enters InService, and rejoins the target group. SNS notifies on-call of the self-healing event; CloudWatch Logs and CloudTrail retain the full audit trail.
